Rule Priority Table — categorization rules above the spending stats

A dedicated Categorization rules table sits directly above the existing spending stats. Each rule has a pattern (text or regex), a target category, and a priority represented by visual order alone — no priority numbers are exposed. Drag rows up or down to change priority; the engine evaluates rules top-down and the first match wins.

  • Drag the ⋮⋮ handle on the left of any row to reorder.
  • Toggle Aa / .* on each row to switch between substring and regex matching.
  • The transactions list at the bottom shows which rule (if any) matched each transaction. Reorder rules to see attributions change.
  • The middle stats table reflects the resulting categorization in real time.

Strengths

  • Priority is implicit in row order — no numeric ranks to hand-edit.
  • Rules, stats, and transaction attribution are all visible together — drag a rule and see the impact instantly.
  • Per-row Aa/.* toggle keeps text and regex rules in one uniform list.

Weaknesses

  • With many rules the table can grow long — no built-in grouping or filtering.
  • Priority via order isn't searchable ("show me priority 12") and is harder to bulk-reorder than numeric ranks.
  • Match count badge can feel noisy; mobile view collapses the per-row layout.
