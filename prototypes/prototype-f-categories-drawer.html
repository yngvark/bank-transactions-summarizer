PROTOTYPE F

Categories Side Drawer

A persistent slide-in panel from the right, mirroring the existing RulesPanel ambient style. The transactions table stays visible while editing — the user sees how changes flow into stats in real time. Edits commit live (no save button); the same beforeunload hint guards unsaved file changes.

  • Click 🗂️ Categories in the toolbar to open / close the drawer.
  • Each primary is a collapsible card. Click the header to expand and edit subcategories.
  • Click the emoji to change it. Double-click any name to rename.
  • Drag handles to reorder primaries or subs within a primary.
  • Search box at top filters visible categories live.

Strengths

  • Stays alongside the data — see edits land in stats immediately.
  • Live save fits the rest of the app (auto-save model).
  • Search makes large taxonomies tractable.

Weaknesses

  • No atomic rollback — every change is final until "undo" toast.
  • Eats horizontal space; on narrow screens it overlays the table.
  • Nested expand/collapse interferes with quick scanning.
