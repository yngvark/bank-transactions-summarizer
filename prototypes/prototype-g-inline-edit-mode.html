Inline Edit-Mode in the Statistics Table — arbitrary depth

An Edit categories toggle pinned above the existing statistics table. While on, every category row at any depth becomes inline-editable. The category tree is now arbitrarily deep — you can add a sub-sub-sub-category, rename it, drag it among its siblings, or delete it (with the whole subtree under it).

  • Toggle ✏️ Edit categories top-right of the statistics section.
  • While ON: drag handles, hover actions, and click-to-rename appear at every depth.
  • Each row exposes + (add child under this node) and × (delete this subtree) on hover.
  • Click an emoji on a top-level category to change it. Press Enter to commit a rename, Esc to cancel.
  • Drag with the ⋮⋮ handle reorders within siblings only.

Strengths

  • Arbitrary depth — sub-sub-…-categories with no model change to the UI.
  • Direct manipulation: edit the thing you see, where you see it.
  • One toggle in/out of edit mode; no separate UI surface to discover.

Weaknesses

  • Deep trees push name columns far right at high depth.
  • Mobile viability remains poor — narrow rows + hover-only actions.
  • "+" affordance is hover-only — discoverability via hint banner alone.
