Inline Edit-Mode in the Statistics Table

An Edit categories toggle pinned above the existing statistics table. While on, every category row becomes inline-editable: emoji picker, rename in place, drag handles to reorder, "+" rows at each level to add. The user edits categories where they're displayed, with no separate UI surface.

  • Toggle ✏️ Edit categories top-right of the statistics section.
  • While ON: drag handles, "+" rows, hover actions, and clickable emoji slots appear.
  • Click a category name to rename. Press Enter to commit, Esc to cancel.
  • Click the emoji to pick a new one (primaries only).
  • Hover a row to reveal the × delete button.

Strengths

  • Direct manipulation — edit the thing you see.
  • No new UI surface or navigation; matches mental model.
  • Categories with no transactions still render in edit-mode (otherwise hidden).

Weaknesses

  • Editing chrome competes with data display — risk of clutter.
  • Mobile viability is poor (small targets, narrow rows).
  • No quick "discard all" — every edit is committed.
