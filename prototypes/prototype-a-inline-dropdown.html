Prototype A

Inline Hierarchical Dropdown

  • Click any category cell to reassign via a 2-level dropdown
  • Uncategorized transactions are highlighted with amber dashed border
  • Hovering an uncategorized cell highlights all rows with the same merchant category
  • Assigning a category shows confirmation that all matching transactions were updated

Pros vs others

  • Fewest clicks per edit — no confirmation step (unlike C)
  • No context switch: stays in the table (unlike B’s panel)
  • Minimal UI chrome — easiest to learn
  • Updates all matching transactions automatically

Cons vs others

  • No search — 2-level dropdown doesn’t scale with many categories (C is searchable)
  • No bulk overview of uncategorized items (B lists them all, D can sort them together)
  • No undo affordance (C has a toast with undo)
  • No rule persistence across sessions (D persists rules in localStorage)
